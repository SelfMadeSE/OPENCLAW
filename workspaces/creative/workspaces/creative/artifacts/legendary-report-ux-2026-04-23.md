## 6. Where to Avoid Overdesign

### Restraint Zones
- **Reading Sections:** Pure, distraction-free text with elegant typography
- **Data Tables:** Clean, functional tables with subtle premium touches
- **Forms:** Simple, focused input areas without decorative elements
- **Navigation:** Clear, intuitive pathways that don't confuse with creativity

### Anti-Patterns to Avoid
- **Animation Overload:** Not everything needs to move. Let content breathe.
- **Complex Interactions:** Don't make users think about how to use the interface.
- **Inconsistent Metaphors:** Stick to one visual language throughout.
- **Performance Traps:** Beautiful but unusable is worse than plain but functional.

### Design Debt Prevention
- **Mobile First:** Ensure premium experience works on all devices
- **Accessibility:** Beauty must not exclude users with disabilities
- **Performance:** All animations must be optimized for smooth rendering
- **Content First:** Design serves the data, not the other way around
